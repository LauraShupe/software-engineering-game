# UseCase Model

## 1. Actors
- Player: A person play the game
- Game System: Runs, manages, and enforces the rules of both stages


## 2. Abstract Use Cases:
- Login
- Play game
- Score Tracker
- Timer
- Restart/Quit Game

## 3. High-Level Use Cases

| Use Case          | Start (TUCBW)                                                      | End / Actor Goal (TUCEW)                                                           |
|-------------------|-------------------------------------------------------------------|------------------------------------------------------------------------------------|
| Login             | TUCBW player creating username/password and logging in            | TUCEW players' game stats are being saved, allowing them to return to their profiles |
| Play Game         | TUCBW player clicking start game                                  | TUCEW player beginning the game                                                    |
| Score Tracker     | TUCBW player viewing leaderboard                                  | TUCEW player sees the high scores                                                  |
| Timer             | TUCBW player is seeing how much time is left                      | TUCEW player is seeing how much time is left                                       |
| Restart/Quit Game | TUCBW player clicking on the restart or quit game button          | TUCEW player restarting the level or quitting the game
